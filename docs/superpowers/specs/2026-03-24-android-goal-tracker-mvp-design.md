# 안드로이드 골 트래커 MVP 설계

## 제품 정의

이 제품은 여러 프로젝트를 동시에 운영하면서, 완료된 프로젝트를 현재 작업의 응원과 근거로 다시 사용하는 안드로이드 우선 개인용 골 트래커다.

출시 목표는 `2026-03-24` 안에 실제 설치 가능한 APK를 만드는 것이다.

## 일정 제약

오늘 안에 설계, 구현, 테스트, 패키징을 끝내야 하므로 범위를 강하게 줄여야 한다.

우선순위:

- 넓은 기능보다 실제로 동작하는 안드로이드 앱
- 외부 연동보다 안정적인 로컬 데이터
- 여러 위젯보다 강한 위젯 1종
- AI 생성보다 단순하고 확실한 템플릿 로직

## 핵심 차별점

대부분의 골 트래커는 할 일, 습관, 프로젝트를 관리한다.  
이 MVP는 아래 한 가지를 증명해야 한다.

`완료된 프로젝트는 레거시로 남고, 현재 프로젝트를 밀어준다.`

## 플랫폼과 기술 스택

- 플랫폼: 안드로이드
- 배포 산출물: APK
- UI: Kotlin + Jetpack Compose
- 저장소: Room 기반 로컬 DB
- 위젯: Android App Widget

## 타깃 사용자

- 개인 안드로이드 사용자
- 여러 개인 프로젝트를 동시에 굴리는 사람
- 실제 내 과거 기록에서 동기부여를 받고 싶은 사람

## MVP 범위

### 1. 홈

홈은 세 가지를 즉시 보여줘야 한다.

- 지금 내가 밀고 있는 포커스 프로젝트가 무엇인지
- 현재 진행 중인 프로젝트가 무엇인지
- 내가 이걸 끝낼 수 있다는 근거가 무엇인지

구성:

- 포커스 프로젝트 카드
- 진행 중 프로젝트 목록
- 완료 프로젝트 기반 응원 문구 1개
- 빠른 진척 확인 진입점

### 2. 프로젝트

프로젝트 관리는 최소 기능만 안정적으로 제공한다.

지원 동작:

- 프로젝트 생성
- 프로젝트 수정
- 프로젝트 완료 처리
- 포커스 프로젝트 지정

지원 상태:

- 진행 중
- 완료

이번 MVP에서 제외:

- 일시정지
- 고급 필터
- 복잡한 태그 UI

### 3. 프로젝트 상세

각 프로젝트는 실제로 운영 가능한 최소 정보만 가진다.

필드:

- 제목
- 의도
- 진행률
- 마일스톤 목록
- 선택 메모

동작:

- 진행률 수정
- 마일스톤 추가
- 완료 처리 시작

### 4. 레거시 저장

프로젝트 완료 시 바로 레거시를 입력받고 저장해야 한다.

필수 입력:

- 이번 프로젝트의 성취 한 줄
- 배운 점 한 줄

선택 입력:

- 대표 결과물
- 상징 문장

### 5. 위젯

MVP는 위젯 1종만 출시한다.

- 포커스 프로젝트 위젯

위젯 구성:

- 포커스 프로젝트 제목
- 진행률
- 레거시가 있으면 짧은 응원 문구

## 응원 엔진

응원 엔진은 결정론적이고 템플릿 기반이어야 한다.

입력:

- 완료 프로젝트 수
- 최근 레거시의 배운 점
- 완료 이력 존재 여부

출력 예시:

- `너는 이미 3개의 프로젝트를 끝냈다.`
- `과거 작업이 이번 프로젝트도 끝낼 수 있다는 증거다.`
- `최근 배운 점: 완벽해지기 전에 먼저 배포하기.`

원칙:

- 한 문장으로 짧게 유지
- 가능하면 사용자가 직접 쓴 레거시를 사용
- 뜬구름 잡는 동기부여 문구는 금지

## 정보 구조

하단 탭:

- 홈
- 프로젝트
- 레거시

핵심 화면:

1. 홈
2. 프로젝트 목록
3. 프로젝트 상세
4. 프로젝트 생성/수정
5. 프로젝트 완료/레거시 저장
6. 레거시 목록

## 디자인 우선순위

UI는 차분하고, 직접적이며, 안드로이드 네이티브스럽게 보여야 한다.

우선순위:

- 프로젝트 스캔 속도
- 포커스 계층의 명확함
- 완료 흐름의 마찰 최소화
- 위젯 가독성

## 팀 분업

### 디자인팀

오늘 기준 담당:

- 안드로이드 비주얼 방향 정의
- 최종 화면 4개 확정
  - 홈
  - 프로젝트 목록
  - 프로젝트 상세
  - 완료/레거시 저장
- 위젯 1종 레이아웃 정의

  - 색상
  - 타이포
  - 간격
  - 카드 스타일
  - 버튼 스타일

핵심은 탐색 폭이 아니라 구현 속도다.

## 성공 기준

오늘 MVP가 성공이라고 볼 조건:

- APK가 실제로 설치된다
- 여러 프로젝트를 생성할 수 있다
- 포커스 프로젝트 1개를 지정할 수 있다
- 프로젝트 완료 시 레거시로 전환된다
- 홈에서 레거시 기반 응원 문구가 보인다
- 위젯이 포커스 프로젝트를 보여준다

## 리스크

1. 위젯은 늦게 붙이면 출시 블로커가 된다.
2. Room과 위젯 갱신 연결은 단순해야 한다.
3. 디자인 과욕은 구현 속도를 떨어뜨린다.

## 결론

가장 먼저 증명해야 하는 건 기능 수가 아니라 이 제품의 관점이다.

`끝난 프로젝트는 사라지지 않는다. 현재 프로젝트를 계속 밀어준다.`
