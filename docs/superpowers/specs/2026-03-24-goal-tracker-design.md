# 골 트래커 설계

## 제품 정의

골 트래커는 여러 프로젝트를 동시에 운영하면서, 완료된 프로젝트를 현재 작업의 동기와 근거로 다시 사용하는 개인용 목표 관리 앱이다.

이 제품은 단순한 습관 앱이나 할 일 앱이 아니다. 핵심은 아래 네 가지다.

- 프로젝트 중심 관리
- 완료 프로젝트의 레거시 보존
- 과거 성취 기반의 응원
- 위젯 중심의 일상 노출

## 문제 정의

기존 목표 앱은 보통 아래 중 하나에 강하다.

- 일일 실행
- 습관과 연속성
- 프로젝트 계획
- 팀 협업

하지만 완료된 프로젝트를 현재의 추진력과 연결하는 경우는 거의 없다.  
대부분 완료 목록으로만 남거나 보관함 뒤로 숨는다.

우리가 노리는 간극은 이거다.

- 완료한 프로젝트가 내 정체성으로 남아야 한다
- 과거의 노력이 현재 프로젝트의 증거가 되어야 한다
- 홈 화면과 위젯이 이 연결을 계속 보여줘야 한다

## 타깃 사용자

v1의 핵심 타깃:

- 개인 사용자
- 여러 프로젝트를 동시에 운영하는 사람
- 할 일보다 프로젝트 맥락을 중요하게 보는 사람
- 생산성 수치뿐 아니라 감정적 보상에도 반응하는 사람

## 제품 원칙

1. 할 일보다 프로젝트
앱의 중심은 프로젝트의 의미와 진척이다.

2. 레거시는 수동 보관이 아니다
완료한 프로젝트는 현재에 계속 영향을 줘야 한다.

3. 위젯은 부가 기능이 아니다
홈 화면에서도 제품의 핵심 가치가 보여야 한다.

4. 10개 이상의 프로젝트도 관리 가능해야 한다
프로젝트 수가 많아져도 복잡하게 무너지면 안 된다.

5. 응원은 실제 근거에서 와야 한다
기계적인 명언보다 사용자의 실제 기록이 중요하다.

## MVP 범위

### 1. 홈

목적:

- 지금의 추진력을 한눈에 보여주기
- 현재 작업과 레거시 응원을 연결하기

구성:

- 포커스 프로젝트
- 진행 중 프로젝트 요약
- 빠른 체크인
- 레거시 응원 문구
- 현재 진행 스냅샷

### 2. 프로젝트 목록

목적:

- 많은 프로젝트를 낮은 마찰로 관리하기

구성:

- 진행 중 / 일시정지 / 완료 구분
- 10개 이상 프로젝트 지원
- 우선순위 / 최근순 / 진행률 정렬
- 빠른 생성

### 3. 프로젝트 상세

목적:

- 각 프로젝트가 단순한 레코드가 아니라 여정처럼 느껴지게 만들기

구성:

- 제목과 의도
- 상태와 기간
- 진행률
- 마일스톤
- 메모
- 하루 집중 기록

### 4. 레거시 보관함

목적:

- 끝난 프로젝트를 의미 있는 자산으로 보존하기

구성:

- 완료 프로젝트 아카이브
- 핵심 성취
- 배운 점
- 대표 결과물
- 상징 문장

### 5. 위젯

목적:

- 앱 바깥에서도 방향성과 동기를 유지하기

초기 위젯 세트:

- 포커스 프로젝트 위젯
- 진행률 링 위젯
- 레거시 응원 위젯

## 핵심 기능

### A. 위젯 시스템

이 앱에서 위젯은 선택 요소가 아니라 핵심 선택 이유다.

요구사항:

- 선택된 포커스 프로젝트 표시
- 현재 진행률 즉시 확인
- 레거시 기반 응원 문구 선택 표시
- 프로젝트 상태 변경 시 빠른 갱신

### B. 레거시 관리

이 기능이 제품의 가장 선명한 차별점이다.

프로젝트가 끝나면 단순 보관이 아니라 레거시 객체가 된다.

각 완료 프로젝트는 아래를 남겨야 한다.

- 무엇을 해냈는지
- 무엇을 배웠는지
- 지금 돌아보면 무엇이 가장 중요한지
- 어떤 결과물이 대표적인지
- 어떤 문장이 상징인지

레거시는 두 가지 방식으로 쓰인다.

- 아카이브 탐색
- 현재 프로젝트 응원 생성

### C. 응원 엔진

응원 엔진은 레거시를 현재 시점의 메시지로 바꾼다.

v1은 AI 생성이 필요 없다. 템플릿 기반이면 충분하다.

입력:

- 완료 프로젝트 수
- 유사 카테고리 경험
- 최근 성취
- 사용자가 직접 쓴 상징 문장

출력:

- 홈 응원 문구
- 프로젝트 상세 응원 문구
- 위젯 응원 문구

원칙:

- 근거 기반이어야 한다
- 일반적인 자기계발 문구처럼 느껴지면 안 된다

### D. 멀티 프로젝트 처리

이 제품은 10개 이상의 프로젝트를 편하게 운영할 수 있어야 한다.

요구사항:

- 진행 중 / 일시정지 / 완료 분리
- 프로젝트 간 빠른 전환
- 단일 목표 강제 금지
- 프로젝트 수가 늘어도 UI가 무너지지 않을 것

v1 권장 모델:

- 포커스 프로젝트 1개
- 진행 중 프로젝트 여러 개
- 일시정지 프로젝트 선택 지원
- 완료 프로젝트는 자동으로 레거시로 이동

## 사용자 흐름

### 신규 사용자

1. 앱 실행
2. 첫 프로젝트 생성
3. 필요하면 여러 프로젝트 추가
4. 포커스 프로젝트 지정
5. 홈과 위젯을 일일 진입점으로 사용

### 완료 흐름

1. 프로젝트 완료
2. 레거시 요약 입력
3. 완료 목록 이동
4. 레거시 레코드 생성
5. 응원 엔진이 해당 기록 사용 가능

### 일상 흐름

1. 사용자가 위젯을 봄
2. 포커스 프로젝트 진입
3. 진행률이나 마일스톤 수정
4. 홈과 응원 문구가 갱신됨

## v1 비범위

- 팀 협업
- 웹 앱
- 안드로이드 동시 출시
- 긴 AI 코칭
- 복잡한 통계 대시보드
- 외부 캘린더/할 일 앱 깊은 연동

## 포지셔닝

Todoist, TickTick, ClickUp, Asana와 비교하면:

- 팀 협업은 약하다
- 엔터프라이즈 프로젝트 기능은 약하다
- 개인 서사와 감정적 연속성은 더 강하다

습관 앱과 비교하면:

- 스트릭 게임화는 약하다
- 프로젝트 정체성과 레거시 의미화는 더 강하다

## 권장 구현 순서

1. 데이터 모델과 로컬 저장
2. 프로젝트 목록과 상세
3. 완료 후 레거시 저장
4. 응원 엔진
5. 위젯
6. 시각적 폴리시와 온보딩

## MVP 성공 기준

- 10개 이상 프로젝트를 혼란 없이 관리 가능
- 포커스 프로젝트 지정 및 전환 가능
- 완료 프로젝트를 레거시로 보존 가능
- 과거 기록이 현재 홈에서 응원으로 작동
- 매일 볼 가치가 있는 위젯 제공

## 리스크

1. 응원 문구가 너무 일반적이면 억지스럽게 느껴질 수 있다.
2. 진행 중 프로젝트가 많을 때 계층이 약하면 혼잡해질 수 있다.
3. 앱 내부 입력이 느리면 위젯 가치가 떨어진다.

## 결론

이 제품의 가장 강한 정의는 넓은 생산성 앱이 아니라 아래 문장이다.

`끝난 프로젝트는 사라지지 않는다. 현재 프로젝트를 계속 밀어준다.`
