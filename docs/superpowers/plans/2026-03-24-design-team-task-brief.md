# 골 트래커 디자인팀 작업지시서

> 목적: 현재 프로젝트 우선 골 트래커의 UX/UI를 실제 제작 가능한 작업 단위로 분해한다.

## 공통 원칙

- 홈은 `현재 포커스 프로젝트`가 가장 먼저 보여야 한다.
- 감성 톤은 유지하되, 정보 우선순위는 매우 선명해야 한다.
- 상태 표시는 `색 + 라벨 + 아이콘`을 함께 쓴다.
- 트래킹은 `히트맵 + 회고형`으로 간다.
- 일 단위 기록은 `했다`만 보이고, 미기록은 빈칸처럼 보여야 한다.

## 작업 우선순위

1. 홈
2. 트래킹
3. 공유 카드
4. 프로젝트 목록
5. 온보딩

## 작업 1: 홈 화면 설계

목표:

- 사용자가 앱을 열자마자 현재 프로젝트를 붙잡게 만든다.

필수 산출물:

- 홈 하이파이 시안 1종
- 포커스 프로젝트 카드 컴포넌트
- 진행 중 프로젝트 흐름 섹션
- 레거시 응원 카드
- `했다` 버튼 상태 시안

체크리스트:

- [ ] 포커스 카드가 화면 최상단에 있다
- [ ] 프로젝트 제목, 진행률, 다음 행동이 1차 정보다
- [ ] 레거시 응원은 보조 카드다
- [ ] `했다` 버튼이 즉시 보인다
- [ ] 카드 밀도가 과하지 않다

완료 기준:

- 리뷰팀이 3초 안에 현재 프로젝트와 다음 행동을 이해할 수 있다고 판단

## 작업 2: 트래킹 화면 설계

목표:

- 시간이 커질수록 사용자가 스스로의 흐름을 보람 있게 읽게 만든다.

필수 산출물:

- 주간 화면
- 월간 히트맵 화면
- 연간 리캡 화면
- 범위 전환 UI
- 빈칸/했다 상태 표현 규칙

체크리스트:

- [ ] 주간은 빠른 리듬 확인이 된다
- [ ] 월간은 히트맵과 리캡이 같이 보인다
- [ ] 연간은 누적 흐름과 완료 회고가 보인다
- [ ] 빈칸이 실패처럼 보이지 않는다
- [ ] 공유 카드와 시각 언어가 연결된다

완료 기준:

- 트래킹이 단순 출석표가 아니라는 점이 시안만 봐도 드러난다

## 작업 3: 공유 카드 설계

목표:

- 사용자가 공유하고 싶을 만큼 보기 좋은 리캡 화면을 만든다.

필수 산출물:

- 주간 공유 카드
- 월간 공유 카드
- 연간 공유 카드

체크리스트:

- [ ] 큰 숫자 1개가 먼저 보인다
- [ ] 대표 문장 1줄이 있다
- [ ] 장식보다 정보가 선명하다
- [ ] 외부 공유 시에도 맥락이 살아 있다

완료 기준:

- 감사팀이 “공유용 요약으로 충분히 자립적”이라고 판단

## 작업 4: 프로젝트 목록 설계

목표:

- 프로젝트가 많아져도 스캔 가능한 구조를 만든다.

필수 산출물:

- 카드형 목록
- 리스트형 목록
- 보기 전환 토글
- 상태 배지 시스템

체크리스트:

- [ ] 카드형이 기본값으로 자연스럽다
- [ ] 리스트형이 고밀도 스캔에 유리하다
- [ ] 두 보기 방식의 의미 구조가 같다
- [ ] 상태가 색+라벨+아이콘으로 보인다

완료 기준:

- 10개 이상 프로젝트를 보여도 혼잡하지 않다

## 작업 5: 온보딩 설계

목표:

- 코칭을 과장하지 않고, 포커스 프로젝트 지정과 첫 `했다` 기록까지 이어지게 만든다.

필수 산출물:

- 시작 인트로
- 프로젝트 선택/생성 화면
- 실행 밀도 선택 화면
- 코칭 보조 톤 선택 화면
- 포커스 확정 및 첫 `했다` 화면

체크리스트:

- [ ] 코치가 주인공처럼 보이지 않는다
- [ ] 온보딩 목적이 프로젝트 선택으로 읽힌다
- [ ] 첫 `했다`까지 흐름이 짧다

완료 기준:

- 온보딩 종료 직후 사용자가 메인 홈으로 자연스럽게 진입한다

## 리뷰 제출 규칙

- 하루 작업분은 화면별 PNG 또는 링크로 제출
- 제출 시 반드시 `목적 / 바뀐 점 / 확인받을 점` 3줄 요약 포함
- 한 번에 여러 화면을 밀어 넣지 말고 우선순위 순으로 제출

## 금지 항목

- 코치 캐릭터를 메인 일러스트 주인공으로 세우기
- 빈칸을 실패 상태처럼 보이게 만들기
- 공유 카드에 정보 과다 적재
- 홈에서 포커스 프로젝트보다 감성 문구를 더 크게 보이게 만들기
