# 골 트래커 구현 티켓 분해안

> 목적: 승인된 UX/UI를 기준으로 디자인팀, 개발팀, 리뷰팀, 감사팀이 같은 단위로 움직일 수 있게 실제 구현 티켓으로 쪼갠다.

## 공통 기준

- 플랫폼은 Android 우선
- 홈의 첫 정보는 포커스 프로젝트
- 일 기록은 `했다` 한 번만 입력
- 미기록은 자동 빈칸
- 앱 전체 시간 기준은 온디바이스 캘린더
- 레거시는 감성 장식이 아니라 현재 프로젝트의 근거
- 전체 카피 톤은 차분하고 공감적인 코칭 톤

## 우선순위 순서

1. 포커스 프로젝트와 일 기록
2. 주/월/년 트래킹 데이터
3. 프로젝트 상세 운영 화면
4. 레거시 전환과 레거시 보관함
5. 연간 리캡과 공유 카드
6. 온보딩 연결

## 티켓 목록

### GT-001 포커스 프로젝트 홈 카드

목표:

- 홈 상단에 포커스 프로젝트를 고정한다

포함:

- 프로젝트 제목
- 상태 라벨/아이콘
- 진행률 수치
- 다음 행동 1줄
- `오늘 했다` 버튼

완료 기준:

- 홈 진입 시 포커스 프로젝트가 가장 먼저 보인다
- `오늘 했다`는 한 번의 탭으로 동작한다

담당:

- 개발팀
- 리뷰팀

### GT-002 홈 프로젝트 흐름 영역

목표:

- 포커스 아래에서 진행 중 프로젝트 흐름을 빠르게 스캔하게 한다

포함:

- 프로젝트 목록 미니 카드
- 카드형/리스트형 상태 반영
- 상태별 색/라벨/아이콘 표시

완료 기준:

- 10개 이상의 프로젝트도 스캔이 가능하다

담당:

- 디자인팀
- 개발팀

### GT-003 일 기록 데이터 모델

목표:

- `했다 / 빈칸` 구조를 저장할 최소 모델을 만든다

포함:

- 프로젝트 ID
- 로컬 날짜
- 기록 여부
- 중복 기록 방지 규칙

완료 기준:

- 같은 날짜에 중복 `했다` 기록이 생기지 않는다
- 미기록은 별도 실패 상태로 저장하지 않는다

담당:

- 개발팀
- 감사팀

### GT-004 주간 흐름 집계

목표:

- 최근 7일 흐름을 프로젝트별로 계산한다

포함:

- 주 시작일 계산
- 7일 배열 생성
- 주말/공휴일 표시 규칙 연결

완료 기준:

- 홈과 상세의 7일 흐름 값이 일치한다

담당:

- 개발팀
- 리뷰팀

### GT-005 월간 실제 달력형 트래킹

목표:

- 월간 뷰를 실제 달력 구조로 렌더링한다

포함:

- 5~6주 달력 셀 생성
- 날짜 좌측 상단 표기
- 주말/공휴일 색상 규칙
- 했다/빈칸/오늘/미래 상태 배경 규칙

완료 기준:

- 가짜 4주 리듬이 아니라 실제 월 구조가 보인다
- 금요일은 기본색, 토요일/일요일/공휴일만 별도 강조된다

담당:

- 개발팀
- 디자인팀
- 감사팀

### GT-006 연간 밀도 집계

목표:

- 1월~12월 월별 밀도 요약을 계산한다

포함:

- 월별 전진 일수 집계
- 완료 프로젝트 수 집계
- 연간 대표 문장용 숫자 제공

완료 기준:

- 연간 화면과 공유 카드가 같은 숫자를 사용한다

담당:

- 개발팀
- 리뷰팀

### GT-007 프로젝트 상세 운영 화면

목표:

- 프로젝트를 실제로 운영하는 중심 화면을 구현한다

포함:

- 프로젝트 이름/상태
- 진행률
- `오늘 했다`
- 다음 행동
- 최근 7일 흐름
- 마일스톤
- 프로젝트의 이유

완료 기준:

- 상세 화면 하나로 프로젝트 운영의 핵심 행동이 가능하다

담당:

- 디자인팀
- 개발팀

### GT-008 마일스톤 관리

목표:

- 프로젝트 진행 맥락을 마일스톤으로 정리한다

포함:

- 마일스톤 생성/수정/완료
- 완료 수 집계
- 상태별 표시

완료 기준:

- 마일스톤 상태가 상세 화면 진행 맥락과 일치한다

담당:

- 개발팀

### GT-009 프로젝트 완료 전환

목표:

- 진행 중 프로젝트를 완료 상태로 전환한다

포함:

- 완료 버튼
- 완료 확인 플로우
- 레거시 입력 진입

완료 기준:

- 완료 직후 레거시 입력 단계로 이어진다

담당:

- 개발팀
- 리뷰팀

### GT-010 레거시 입력 플로우

목표:

- 완료한 프로젝트의 핵심 흔적을 짧게 남긴다

포함:

- 남긴 일
- 남은 말
- 대표 결과물

완료 기준:

- 사용자는 길게 쓰지 않아도 레거시를 남길 수 있다

담당:

- 디자인팀
- 개발팀

### GT-011 레거시 보관함

목표:

- 완료 프로젝트를 다시 보기 좋은 형태로 모아둔다

포함:

- 완료 프로젝트 목록
- 각 항목의 3축 정보
- 완료 개수 요약

완료 기준:

- 완료 프로젝트가 단순 목록이 아니라 기억 가능한 카드로 보인다

담당:

- 디자인팀
- 개발팀

### GT-012 현재 프로젝트 근거 카드

목표:

- 레거시가 현재 프로젝트에 실제 근거로 연결되게 한다

포함:

- 완료 프로젝트 수 기반 문장
- 비슷한 프로젝트 연결 문장
- 최근 남은 말 재사용

완료 기준:

- 홈/레거시/상세 어디서 봐도 과장 없는 근거 문장만 보인다

담당:

- 개발팀
- 감사팀

### GT-013 월간 리캡 카드

목표:

- 실제 달력 위의 기록을 한 줄과 숫자로 요약한다

포함:

- 이번 달 전진 일수
- 최대 흐름
- 한 줄 요약 문장

완료 기준:

- 월간 화면과 공유 카드가 동일한 기준 데이터에서 나온다

담당:

- 개발팀
- 리뷰팀

### GT-014 연간 공유 카드

목표:

- 연간 누적감을 예쁜 카드로 요약한다

포함:

- 큰 한 줄 문장
- 핵심 숫자 2개
- 월별 흐름 압축 표시

완료 기준:

- 화면 캡처만으로도 의미가 읽힌다

담당:

- 디자인팀
- 개발팀

### GT-015 카피 톤 시스템

목표:

- 앱 전반 문구를 같은 톤으로 유지한다

포함:

- 홈 문구
- 상세 문구
- 레거시 문구
- 리캡/공유 문구

완료 기준:

- 다나까식 안내문이 사라진다
- 강요하거나 과장하는 문장이 없다

담당:

- 디자인팀
- 기획팀
- 리뷰팀

### GT-016 온보딩 연결

목표:

- 온보딩 종료 후 첫 기록까지 바로 이어지게 한다

포함:

- 프로젝트 선택 또는 생성
- 포커스 지정
- 첫 `했다` 기록
- 홈 진입 연결

완료 기준:

- 온보딩을 끝내면 빈 상태가 아니라 첫 기록이 있는 상태로 진입한다

담당:

- 개발팀
- 리뷰팀

## 리뷰팀 확인 항목

- 홈에서 핵심 행동이 한 탭인지
- 월간이 실제 달력처럼 읽히는지
- 금요일이 잘못 강조되지 않는지
- 레거시 문장이 데이터 근거를 벗어나지 않는지
- 카피 톤이 압박적이지 않은지

## 감사팀 확인 항목

- 미기록이 실패 상태로 저장되지 않는지
- 캘린더 기준이 화면마다 충돌하지 않는지
- 레거시가 감성 장식으로 과장되지 않는지
- 공유 카드 숫자와 원본 데이터가 일치하는지

8. GT-012 현재 프로젝트 근거 카드
